Rules & Regulations

Developer Rules
  1. Make sure all code is in https://gitlab.myteksi.net/engineering/grabathon/grabathon, create a sub folder under Oct2018 with your team code. You can upload it close to finishing to avoid any merge conflicts / accidentally deleting someone else's code.
  2. Do not make any modifications on staging or production, or teams will face disqualification.
  3. Do not use existing staging / production keys if you are using 3rd party tools like Twilio / Nexmo / Google etc.
  4. You can use your team's sub account for projects but
    • Remember to get approval from the owner of the subaccount that you have access to. These should be developer test sub accounts not sub accounts like Data Engineering which has production workloads.
    • Remember to destroy all instances after they are used
    • Remember to keep the usage below $25
  5. Existing APIs on staging are accessible for your projects. Alternatively, you might want to check out Marionette (http://marionette.stg-myteksi.com/) and Marionette API (https://wiki.grab.com/x/OB3WCg) by the Internal Tools team.
