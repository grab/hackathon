Developers Rules

  • Make sure all code is in the grabathon repository. Create a sub folder under Oct2017 with your team code. You can upload it close to finishing to avoid any merge conflicts / accidentally deleting someone else's code.
  • Do not make any modifications on staging or production, or teams will face disqualification.
  • Do not use existing staging / production keys if you are using 3rd party tools like Twilio / Nexmo / Google etc.
  • You can use your team's sub account for projects but remember to get approval from the owner of the subaccount that you have access to. These should be developer test sub accounts not sub accounts like Data Engineering which has production workloads.
  • Destroy all instances after they are used.
  • Keep your AWS usage under SG$25 per team.
  • Existing APIs on staging are accessible for your projects. Alternatively, you might want to check out Marionette and Marionette API by the Internal Tools team.

For queries and questions please reach out via the official Slack channel #grabathon.
